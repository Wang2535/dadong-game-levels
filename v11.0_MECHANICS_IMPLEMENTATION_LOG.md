## 三、骰子判定机制

### 3.1 难度等级

| 难度 | 失败点数 | 成功率 | 描述 |
|------|----------|--------|------|
| 1 | 1 | 83.3% | 简单 |
| 2 | 1-2 | 66.7% | 普通 |
| 3 | 1-3 | 50.0% | 困难 |
| 4 | 1-4 | 33.3% | 极难 |
| 5 | 1-5 | 16.7% | 极限 |
| 6 | 任意 | 0% | 不可能 |

### 3.2 特殊判定

- **大成功**: 掷出6且难度≤3
- **大失败**: 掷出1且难度≥4

### 3.3 API

```typescript
// 基础判定
DiceMechanic.roll({ baseDifficulty: 3 })

// 带修正判定
DiceMechanic.roll({ baseDifficulty: 4, difficultyModifier: 1 })

// 带重掷判定
DiceMechanic.rollWithReroll({ baseDifficulty: 3, rerollCount: 1, takeBest: true })
```

## 四、剪刀石头布机制

### 4.1 胜负规则

| 发起方 | 响应方 | 结果 |
|--------|--------|------|
| 剪刀 | 布 | 发起方赢 |
| 布 | 石头 | 发起方赢 |
| 石头 | 剪刀 | 发起方赢 |
| 相同 | 相同 | 平局 |

### 4.2 效果定义

卡牌可定义三种结果的效果：
- **平局效果** (drawEffect)
- **赢方效果** (winEffect)
- **输方效果** (loseEffect)

### 4.3 API

```typescript
// 执行对决
RPSMechanic.resolve('scissors', 'paper', 'player1', 'player2')

// 获取效果
RPSMechanic.getEffect(result, config, initiatorId, responderId)

// AI随机选择
RPSMechanic.randomChoice()
```

## 五、7阶段回合系统

### 5.1 阶段流程

```
判定阶段 → 恢复阶段 → 摸牌阶段 → 行动阶段 → 响应阶段 → 弃牌阶段 → 结束阶段
```

### 5.2 阶段配置

| 阶段 | 自动推进 | 时间限制 | 可用卡牌 | 可用技能 |
|------|---------|---------|---------|---------|
| 判定 | ✅ | - | ❌ | ❌ |
| 恢复 | ✅ | - | ❌ | ❌ |
| 摸牌 | ✅ | - | ❌ | ❌ |
| 行动 | ❌ | 60秒 | ✅ | ✅ |
| 响应 | ❌ | 10秒 | ✅ | ✅ |
| 弃牌 | ❌ | 15秒 | ❌ | ❌ |
| 结束 | ✅ | - | ❌ | ❌ |

### 5.3 资源恢复（v11.0调整）

**每回合恢复**: 信息+2，算力+2，资金+2

```typescript
FACTION_RECOVERY = {
  attack: { actionPoints: 3, compute: 2, funds: 2, information: 2 },
  defense: { actionPoints: 3, compute: 2, funds: 2, information: 2 }
}
```

### 5.4 摸牌配置

**保留原有**: 每回合抽4张

```typescript
DRAW_PHASE_CONFIG = {
  baseDraw: 4,
  extraDrawWhenEmpty: 0
}
```

### 5.5 弃牌配置

**手牌上限**: 1张

```typescript
DISCARD_PHASE_CONFIG = {
  maxHandSize: 1
}
```

// 执行难度3判定

// 执行对决
