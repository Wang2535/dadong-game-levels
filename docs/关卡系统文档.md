## 系统概述

《大东话安全》是一款以网络安全为主题的策略卡牌桌游。游戏通过9个精心设计的关卡，让玩家体验从计算机病毒到账号安全的各种网络安全场景。

### 核心设计理念
- **教育性**：每个关卡都基于真实的网络安全案例
- **趣味性**：结合电影、历史事件等通俗元素
- **策略性**：需要合理分配资源、选择卡牌、应对敌人

### 游戏流程
```
开始 → 选择关卡 → 配置资源 → 回合开始 → 大东AI行动 → 玩家行动 → 敌人行动 → 回合结束 → 判定胜负
```

## 敌人角色

### 敌人分类

游戏中共包含18个敌人角色，分为9个关卡，每关2个敌人。

#### 第1关：病毒初现

**埃尔克克隆者 (Elk Cloner)**
- **类型**：病毒型敌人
- **攻击方式**：潜伏复制、信息干扰
- **弱点**：接种签名、系统重写
- **技能**：
  - 软盘感染：放置标记时感染区域，下回合友方标记-1
  - 第50次启动：累计5个标记触发，友方角色下回合行动点-1

**传播者斯克伦塔 (Skrenta the Spreader)**
- **类型**：黑客型敌人
- **攻击方式**：主动传播、资源窃取
- **弱点**：安全意识、及时更新
- **技能**：
  - 社交工程传播：将友方标记转化为敌方标记
  - 潜伏复制：受攻击时有50%概率复制标记

#### 第2关：AI的抉择

**叛逆莫斯 (Rebel MOSS)**
- **类型**：人工智能型敌人
- **攻击方式**：逻辑判断、控制权争夺
- **弱点**：物理破坏、人工干预
- **技能**：
  - 底层命令优先：友方标记少于敌方时，友方标记-1
  - 计算否定：掷骰子≥4阻止操作，下回合行动点-1

**AI攻击者 (AI Attacker)**
- **类型**：智能攻击型敌人
- **攻击方式**：自动化攻击、效率倍增
- **弱点**：行为异常检测、安全意识
- **技能**：
  - 自动化扫描：每回合扫描所有区域放置标记
  - 鱼叉式钓鱼：迫使玩家弃牌或行动点-2

#### 第3关：蠕虫危机

**熊猫烧香 (Panda Burning Incense)**
- **类型**：蠕虫病毒型敌人
- **攻击方式**：快速传播、高调感染
- **弱点**：杀毒软件、系统补丁、安全意识
- **技能**：
  - 自我繁殖：放置标记后立即在相邻区域复制
  - 高调感染：敌方标记达到3个时，友方标记转为敌方标记

**网络蠕虫 (Network Worm)**
- **类型**：蠕虫型敌人
- **攻击方式**：漏洞扫描、现场处理
- **弱点**：入侵检测、系统加固、及时响应
- **技能**：
  - 漏洞扫描：每回合扫描所有区域放置标记
  - 现场处理：标记被移除时有30%概率留下后门

#### 第4关：组件化攻击

**火焰病毒 (Flame Virus)**
- **类型**：高级持续性威胁（APT）型敌人
- **攻击方式**：组件化加载、隐蔽渗透
- **弱点**：病毒检测工具、系统备份、及时响应
- **技能**：
  - 组件化加载：选择感知/攻击/隐藏模块执行效果
  - 自我删除：受攻击时转移标记到其他区域

**震网病毒 (Stuxnet)**
- **类型**：工业控制系统攻击型敌人
- **攻击方式**：零日漏洞、工业渗透
- **弱点**：系统补丁、安全防护、多层防御
- **技能**：
  - 零日漏洞攻击：友方标记-2，下回合行动点消耗+1
  - 工业控制渗透：敌方标记≥3时区域进入失控状态

#### 第5关：致命缺陷

**致命Bug (Fatal Bug)**
- **类型**：软件缺陷型敌人
- **攻击方式**：传感器误导、控制权争夺
- **弱点**：系统更新、人工干预、多源验证
- **技能**：
  - 传感器误导：友方标记下一回合被误判为敌方标记
  - 控制权争夺：敌方≥友方时，区域进入失控俯冲状态

**设计缺陷 (Design Flaw)**
- **类型**：系统漏洞型敌人
- **攻击方式**：故障触发、指令冲突
- **弱点**：全面测试、冗余设计、安全培训
- **技能**：
  - 故障模式触发：友方使用卡牌时友方标记-2
  - 指令冲突：掷骰子偶数则最后一个操作无效

#### 第6关：工控危机

**黄金眼卫星 (GoldenEye Satellite)**
- **类型**：工控攻击型敌人
- **攻击方式**：电磁脉冲、远程干扰
- **弱点**：物理防护、系统隔离、备份恢复
- **技能**：
  - 电磁脉冲攻击：区域瘫痪，友方无法使用卡牌2回合
  - 远程控制干扰：友方标记无法移动到其他区域

**工控入侵者 (ICS Intruder)**
- **类型**：基础设施攻击型敌人
- **攻击方式**：基础设施渗透、连锁故障
- **弱点**：终端保护、控制器加固、安全监测
- **技能**：
  - 基础设施渗透：渗透区域每回合友方标记-1
  - 连锁故障：相邻区域都有标记时触发连锁反应

#### 第7关：隐私透明

**探针盒子 (Probe Box)**
- **类型**：隐私窃取型敌人
- **攻击方式**：MAC探测、数据关联
- **弱点**：关闭WiFi、MAC地址随机化、隐私意识
- **技能**：
  - MAC地址探测：获得信息碎片，3个时放置标记
  - 数据关联：移除友方标记，获得精准画像状态

**隐私窃贼 (Privacy Thief)**
- **类型**：数据窃取型敌人
- **攻击方式**：权限滥用、后台监听
- **弱点**：权限管理、隐私意识、数据加密
- **技能**：
  - 权限滥用：移除友方标记，获得隐私卡
  - 后台监听：查看手牌并选择一张使其失效

#### 第8关：密码之战

**希瓦纳博士 (Dr. Sivana)**
- **类型**：密码破解型敌人
- **攻击方式**：密码分析、七宗罪之力
- **弱点**：强密码、多因素认证、加密算法更新
- **技能**：
  - 密码分析：掷骰子≥4则友方标记被解密
  - 七宗罪之力：成功解密后技能效果+1

**密码破译者 (Code Breaker)**
- **类型**：密码分析型敌人
- **攻击方式**：暴力破解、历史攻击
- **弱点**：现代加密算法、密钥管理、安全协议
- **技能**：
  - 暴力破解：消耗行动点掷骰子，6点则失去标记
  - 历史攻击：标记数≥友方时，友方行动点消耗+1

#### 第9关：账号保卫战

**盗号黑手 (Account Thief)**
- **类型**：账号窃取型敌人
- **攻击方式**：钓鱼陷阱、撞库攻击
- **弱点**：强密码、异常检测、设备绑定
- **技能**：
  - 钓鱼陷阱：设置陷阱，触发时标记转为敌方标记
  - 撞库攻击：拥有账号信息时移除友方标记

**做号集团 (Account Farm)**
- **类型**：黑产组织型敌人
- **攻击方式**：批量产号、养号运营
- **弱点**：实名制验证、设备指纹、行为分析
- **技能**：
  - 批量产号：每回合自动在随机区域放置标记
  - 养号运营：标记数≥3时，每回合获得1行动点

### 卡牌编号规则

**XZYa-bC 格式**：
- **X** = 来源类 (S角色特殊/N正常/L关卡)
- **Z** = 功能类 (E资源交换/F基础数值/I即时判定/A延时判定/D数值降低/O其他)
- **a** = 关卡数 (0表示现有卡牌，1-9表示关卡)
- **b** = 卡牌序号
- **C** = 科技等级 (T1-T5)

## 区域系统

### 四区域架构

每个关卡包含4个区域，每个区域有不同的特性和初始标记分布。

#### 区域类型

1. **internal（内部网络）**
   - 通常是核心区域
   - 敌方技能效果可能增强
   - 友方可以使用特殊卡牌

2. **industrial（工业控制）**
   - 工控系统专用区域
   - 失控会影响其他区域
   - 需要特别保护

3. **dmz（隔离区）**
   - 中立区域
   - 可能有传播特性
   - 双方都可以利用

4. **external（外部网络）**
   - 安全区域
   - 敌方技能效果减弱
   - 友方标记受保护

### 区域控制规则

- **攻击标记 > 防御标记**：区域被敌方占领
- **防御标记 > 攻击标记**：区域被玩家占领
- **标记相等**：区域中立

## 游戏规则

### 回合流程

1. **回合开始**
   - 恢复行动点
   - 触发回合开始效果

2. **大东AI行动**
   - 恢复行动点（4点）
   - 抽牌（2张）
   - 根据资源选择可打的卡牌
   - 资源恢复（各+3）

3. **玩家行动**
   - 使用行动点打出卡牌
   - 移动标记
   - 使用技能

4. **敌人行动**
   - 根据攻击模式执行行动
   - 触发被动技能
   - 放置攻击标记

5. **回合结束**
   - 判定区域控制
   - 检查目标完成
   - 触发回合结束效果

### 胜利条件

每个关卡有3个目标，完成任意1个即可胜利：

1. **清除敌人**：所有区域敌方标记总数降至0
2. **区域控制**：友方标记达到特定条件
3. **使用特殊卡牌**：使用关卡特殊卡牌达成效果

### 失败条件

满足以下任一条件即失败：

1. **安全等级归零**：玩家安全等级降至0
2. **回合超限**：超过关卡回合限制
3. **核心区域沦陷**：关键区域被敌方完全控制

## 技术实现

### 文件结构

```
src/
├── data/
│   ├── levelDatabase.ts      # 关卡数据库
│   ├── levelEnemies.ts       # 敌人角色数据库
│   ├── levelCardDatabase.ts  # 关卡解锁卡牌数据库
│   └── cardDatabase.ts       # 基础卡牌数据库
├── types/
│   ├── levelTypes.ts         # 关卡类型定义
│   └── legacy/card_v16.ts    # 卡牌类型定义
└── engine/
    └── LevelGameStateManager.ts  # 关卡游戏状态管理
```

### 核心类型

#### LevelDefinition
关卡定义接口，包含关卡的所有配置信息。

#### EnemyCharacter
敌人角色接口，包含敌人的属性、技能、背景等。

#### AreaDistribution
区域分布接口，定义每个区域的名称、初始标记、特性。

### 数据关联

```
LevelDefinition
├── enemyIds → LevelEnemies
├── rewards.unlockedCards → LevelCardDatabase
├── areaDistribution → AreaDistribution
└── specialCards → LevelSpecialCard
```
